## 笔记整理

<p>Skills 是 Claude Code 命令体系的下一代架构——它不只是&quot;更好的自定义命令&quot;，而是一套完整的能力 封装标准，支持自动触发、子 Agent、动态上下文注入，并已成为 30+ AI 工具共同采纳的开放标准。</p>

<p>本节深入解析 SKILL.md 的 15 个 Frontmatter 字段、触发机制的工作原理，以及 agentskills.io 生态。</p>

<h2>§19.1 SKILL.md 基本结构</h2>

<p>每个 Skill 是一个文件夹，核心是 SKILL.md 文件：</p>

<p>--- description: 技能描述（触发机制核心） --- # Skill 正文 Markdown 格式的执行指令... 完整的 Skill 目录结构：</p>

<pre><code>├── SKILL.md ← 主指令文件（必须）</code></pre>

<pre><code>├── template.md ← Claude 填写的模板（可选）</code></pre>

<pre><code>├── examples/</code></pre>

<pre><code>│ └── sample.md ← 期望输出的示例（可选）</code></pre>

<pre><code>├── references/</code></pre>

<pre><code>│ └── api-docs.md ← 按需加载的参考文档（可选）</code></pre>

<pre><code>└── scripts/</code></pre>

<pre><code>└── validate.sh ← Claude 可执行的脚本（可选）</code></pre>

<h2>§19.2 三级渐进加载系统</h2>

<p>Skills 采用三级加载机制，平衡上下文用量与功能完整性：</p>

<p>加载级别 | 内容 | 何时加载</p>

<p>元数据 name + description（约 100 词） 始终在上下文中</p>

<p>SKILL.md 正文 | 完整指令（建议 &lt;500 行） | Skill 被触发时</p>

<p>辅助资源 references/、scripts/ 等按需，Claude 判断后决定加载</p>

<h2>§19.3 全部 15 个 Frontmatter 字段详解</h2>

<p>是否 字段 说明 必需 name 推荐 Skill 标识符，成为斜杠命令名。小写字母+数字+连字符，最长 64 字符。</p>

<p>省略时使用目录名 描述功能和触发时机。Claude 根据此字段决定是否自动加载。省略时使 description 强烈 用正文第一段 推荐 argument-hint 否自动补全时显示的参数提示，如 [issue-number] 或 [filename]</p>

<p>否 | true 时禁止 Claude 自动触发，只允许用户手动调用（适合</p>

<p>/commit、/deploy 等需精确控制时机的命令）。默认 false user-invocable 否 false 时从 / 菜单隐藏，只有 Claude 可以调用（适合背景知识型 Skill）。默认 true allowed-tools 否此 Skill 激活时 Claude 可免确认使用的工具列表。如 Bash(gh *), model 否此 Skill 激活时使用的模型，覆盖会话设置 effort 否此 Skill 激活时的 Effort 等级，覆盖会话设置。选项：low 、medium 、 high 、max （仅 Opus 4.6） context 否设为 fork 时在隔离的子 Agent 上下文中运行，适合独立的研究/分析任 务 agent 否 context: fork 时使用的 Agent 类型：Explore （只读探索）、 Plan （规划）、general-purpose （通用）或自定义 Agent hooks 否作用域限于此 Skill 生命周期的 Hooks 配置 compatibility 否所需工具、依赖声明（较少使用） $ARGUMENTS （变量） — 调用时传入的全部参数。SKILL.md 中无此占位符时，参数追加到末尾</p>

<p>$ARGUMENTS[N] （变量） | — | 按 0 索引访问特定参数，简写 $0 、$1 、$2</p>

<p>— | 包含 SKILL.md 的目录路径，用于引用捆绑脚本，确保路径与工作目录无</p>

<p>（变量） 关</p>

<h2>§19.4 触发机制深度解析</h2>

<p>description 是自动触发的核心。Claude 将所有 Skill 的 name + description 加载到上下文，根据用户输入决定 是否调用：</p>

<p>不推荐 推荐 过于被动的 description（Claude 难以触发） 积极主动的 description（包含触发词和场景） description: 代码解释功能 用可视化图表和类比解释代码工作原理。</p>

<p>当用户询问&quot;这段代码怎么运行&quot;、&quot;解释一下这个文 件&quot;、 &quot;这里的逻辑是什么&quot;时使用。</p>

<p>即使用户只说&quot;这是什么意思&quot;也应触发。</p>

<p>调用控制矩阵：</p>

<p>Frontmatter 设置用户可调用 Claude 可自动触发 description 是否在上下文</p>

<p>默认 | 是 | 是 | 始终在上下文</p>

<p>disable-model-invocation: true 是否不在上下文（完全手动） user-invocable: false 否是始终在上下文</p>

<h2>§19.5 动态上下文注入（Shell 预处理）</h2>

<p>使用 !`&lt;命令&gt;` 语法，在 Skill 内容发送给 Claude 之前执行 Shell 命令，将输出直接注入到提示中：</p>

<p>--- description: 总结 Pull Request 的变更 --- ## PR 上下文 - 变更文件：!`gh pr diff --name-only` ## 你的任务 总结这个 Pull Request，指出主要变更和潜在风险。</p>

<p>注意 这是预处理，不是 Claude 执行命令。Shell 命令在 SKILL.md 渲染时就已执行，Claude 只看到注入后的最终 文本。这意味着注入内容对 Claude 而言是&quot;已知事实&quot;，而非&quot;待执行任务&quot;。</p>

<h2>§19.6 高级模式：子 Agent 与扩展思考</h2>

<p>在子 Agent 中运行（context: fork） context: fork 会创建隔离上下文，Skill 内容成为子 Agent 的任务提示。子 Agent 没有当前对话历史的访问 权限，适合独立的分析任务：</p>

<p>--- description: 深度调研某个主题 --- 深度调研 $ARGUMENTS：</p>

<p>1. 使用 Glob 和 Grep 寻找相关文件 2. 阅读并分析代码 3. 以具体文件引用总结发现 开启扩展思考 在 SKILL.md 内容中任意位置包含 ultrathink 即可开启扩展思考模式，让 Claude 在执行前深度推理：</p>

<p>--- description: 设计复杂系统架构 --- 请设计以下系统的架构：$ARGUMENTS</p>

<h2>§19.7 Skills 权限控制</h2>

<p>在 /permissions 中可以精细控制哪些 Skills 可以自动触发：</p>

<p># 禁用所有 Skill 的自动触发 # 只允许特定 Skill 触发 # 拒绝特定高风险 Skill 的自动触发 权限语法：Skill(name) 精确匹配，Skill(name *) 前缀匹配加任意参数。</p>

<h2>§19.8 agentskills.io 开放标准</h2>

<p>Claude Code Skills 遵循 agentskills.io 开放标准，最初由 Anthropic 开发，现已被 30+ AI 工具采纳，包括：</p>

<p>Cursor · GitHub Copilot · VS Code · Gemini CLI · OpenCode · Roo Code · OpenHands 这意味着：你在 Claude Code 中编写的 Skill，理论上可以在以上所有工具中直接复用。Claude Code 的扩展字 段（如 context: fork 、agent: Explore ）在不支持的工具中会被忽略，核心功能依然生效。</p>

<p>分发方式 | 路径 | 适用对象</p>

<p>打包分发 | .skill 文件 | 公开分享</p>
