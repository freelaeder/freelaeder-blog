## 笔记整理

<p>Claude Code 的能力远不止&quot;帮你写代码&quot;。它拥有一套完整的工具体系，覆盖文件操作、代码执行、 Terminal 管理和 Git 工作流。本节通过能力矩阵，帮你建立对 Claude Code 全貌的系统认知。</p>

<p>能力矩阵：四大核心领域</p>

<p>能力领域 | 具体能力 | 核心工具 | 典型场景</p>

<p>文件操作 | 读取任意文件（含 PDF、图片、</p>

<p>Read 理解现有代码、分析配置 精确字符串替换 Edit Bug 修复、局部重构 创建/完整覆盖文件 Write 新建模块、生成测试文件 文件模式搜索 Glob 查找特定类型文件、探索结构 代码执行运行测试套件 Bash pytest / npm test / go test 构建与编译 Bash npm build / make / cargo 运行任意脚本 Bash 数据处理、数据库迁移 Terminal 系统状态查询 Bash 查看进程、环境变量 CLI 工具调用 Bash gh / aws / kubectl / docker 内容搜索（正则） Grep 跨文件查找函数定义、追踪引 用 网络请求 WebFetch / 获取最新文档、搜索解决方案 Git 集成查看状态与差异 Bash (git) git status / diff / log 创建提交（含自动生成 commit Bash (git) git commit / push 创建 Pull Request Bash (gh) gh pr create / review 并行工作树 Bash (git worktree) 多任务并行、不同功能隔离 文件操作：读、写、改的精确控制 Claude Code 对文件的操作遵循一个明确的优先级原则：Edit 优于 Write，Read 先于 Edit。</p>

<p>不推荐 推荐 修改现有文件时，优先使用 Edit 不要对现有文件直接使用 Write Edit 工具接受 old_string （原文本）和 Write 工具会完整覆盖文件内容，无法做合并。在现有 文件上使用 Write 容易丢失其他内容，且 diff 不清晰， new_string （新文本），只修改指定部分，展示清晰 的 diff 供审阅。如果 old_string 在文件中不存在， 难以 review。</p>

<p>会直接报错而非静默失败，安全性更高。</p>

<p>文件读取支持分段处理。对于大型文件（如几千行的源码文件），使用 offset 和 limit 参数可以只读取需要 的部分，节省上下文窗口：</p>

<p># Claude Code 内部的 Read 调用示例（你无需手动写这些） # 只读取第 100-150 行，精准定位 代码执行：Bash 工具的完整能力 Bash 工具是 Claude Code 与系统交互的主要通道。它不是一个受限的沙盒，而是真实的 Shell 环境。这意味着 Claude Code 可以做任何你在终端里可以做的事情：</p>

<p>测试 pytest tests/ -v --tb=short 运行测试套件，查看详细输出 构建 npm run build 、cargo build --release 编译项目，检查构建错误 依赖管理 pip install -r requirements.txt 安装项目依赖 数据库 python manage.py migrate 执行数据库迁移 云服务 aws s3 ls 、kubectl get pods 管理云资源 代码质量 eslint src/ 、black . 运行 linter / formatter 注意 Bash 工具默认需要你的批准才会执行，尤其是具有副作用的命令（如文件删除、网络请求）。你可以在 .claude/settings.json 中为已知安全的命令设置白名单，减少重复确认的摩擦。</p>

<p>Git 集成：天生的 Git 原住民 Claude Code 对 Git 的理解超出了大多数人的预期。它不只会执行 git 命令，更能理解 git 的上下文语义，做出 符合工程实践的决策。</p>

<p>一个典型的 Claude Code + Git 工作流如下：</p>

<p>1 | 探索与规划</p>

<p>在动手前，Claude Code 会先运行 git log --oneline -20 、git diff main 等命令，了解当前分支 状态、最近的提交历史和待合并的变更。</p>

<p>2 | 实现功能</p>

<p>修改代码时，Claude Code 自动在每次 Edit/Write 前创建 Checkpoint，允许你随时按两下 Esc 或使用 /rewind 命令回滚。</p>

<p>3 | 自动生成提交信息</p>

<p>完成实现后，Claude Code 会查看 git diff --staged，理解变更的语义，生成一条符合 Conventional Commits 规范的提交信息（而非&quot;fix bug&quot;这种无意义的提交）。</p>

<p>4 创建 Pull Request 通过 gh pr create，Claude Code 可以创建包含详细描述的 PR，自动填充变更摘要、测试计划、相关 issue 链接等信息。</p>

<p>工具选择决策逻辑 Claude Code 内部有一套工具选择的优先级逻辑，理解这套逻辑有助于你理解它的行为模式：</p>

<p>验证结果 需要了解代码？</p>

<p>需要修改代码？</p>

<p>需要执行操作？</p>

<p>Read / Grep / Glob →</p>

<p>Edit（局部）/ Write（全新） | →</p>

<p>Bash → Bash（测试） 核心建议 Claude Code 的内置规则要求它优先使用专用工具（Read / Grep / Glob），而非通过 Bash 执行 cat / grep / ls。专用工具有更好的权限优化和格式化输出，你在日志中看到的工具调用能反映这一点。</p>
