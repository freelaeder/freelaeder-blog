## 笔记整理

<p>面对数十万行的大型代码库，如何让 Claude 快速定位到正确的文件，而不是消耗大量上下文进行无效 搜索？三层工具体系提供了系统性的解决方案。</p>

<p>三层工具体系 Claude Code 提供三类导航工具，构成从轻到重的三层体系。核心策略是：Glob/Grep 先行，Read 按需——先 用轻量工具将候选范围压缩到 3-5 个文件，再对最相关的文件进行深度阅读。</p>

<p>层次 | 工具 | 机制 | Token 成本 | 适用场景</p>

<p>第一层 Glob 文件名模式匹配极低找文件在哪里 第二层 Grep 内容正则搜索低找代码在哪里 第三层 Read 读取完整文件内容中高理解具体逻辑 第一层：Glob——找文件位置 Glob 是按文件名模式匹配的最轻量工具，结果按修改时间排序，权限处理优于 Bash 的 find 命令。</p>

<p>常用 Glob 模式：</p>

<p>模式 | 含义 | 典型使用场景</p>

<p>**/*.ts 任意目录下的所有 TypeScript 文件了解项目的 TS 文件分布 **/*.test.ts 所有测试文件查看测试覆盖范围 src/**/* src 目录下的所有文件探索主要源码结构 **/package.json 所有包配置文件了解 monorepo 结构 **/*.{js,jsx} 所有 JS/JSX 文件排查前端代码分布 **/CLAUDE.md 所有项目配置文件了解各子模块的约定 Prompt 示例——使用 Glob 探索结构：</p>

<p>用 Glob 列出 src/ 目录下所有的 *.service.ts 文件， 然后给我一个这些服务类的高层概述——每个服务的职责是什么。</p>

<p>这个 monorepo 有多少个子包？</p>

<p>用 Glob 找到所有的 package.json，列出每个包的名称和描述。</p>

<p>第二层：Grep——找代码位置 Grep 基于 ripgrep 实现，支持完整的正则表达式语法，是在大型代码库中精确定位代码的利器。三种输出模式 各有用途：</p>

<p>输出模式 | 返回内容 | 适用场景</p>

<p>files_with_matches （默认） | 匹配的文件路径列表 | 确认哪些文件包含目标代码</p>

<p>content 匹配行及行号（支持上下文） 查看具体的代码片段 count 每个文件的匹配次数评估引用范围 Prompt 示例——精确定位代码：</p>

<p>用 Grep 搜索 &quot;authenticateToken&quot;（大小写不敏感）， 输出模式用 content，显示每个匹配行的前后 3 行上下文。</p>

<p>只搜索 *.ts 文件。</p>

<p>找到所有使用 UserService 的文件（Grep: import.*UserService）， 然后按调用频次排序，告诉我哪个模块对 UserService 的依赖最深。</p>

<p>搜索所有包含 &quot;TODO&quot; 注释的 Python 文件（type: py）， 按文件路径分组列出，帮我评估技术债务的分布情况。</p>

<p>第三层：Read——理解具体逻辑 Read 工具读取文件完整内容（附行号），是消耗 token 最多的工具，但也是理解复杂逻辑的唯一手段。关键使 用原则：</p>

<p>大型代码库的专项策略 策略一：分而治之 对于超大代码库（10 万行以上），采用模块化分析策略，避免一次性载入过多上下文：</p>

<p># 第一步：了解全局结构 列出这个项目的主要模块及其职责（用 Glob 扫描顶层目录） # 第二步：聚焦单个模块 专注于 src/payment/ 模块，详细解释其内部架构和主要类 # 第三步：追踪跨模块调用链 追踪一次支付请求从 API 入口到数据库写入的完整流程， 只关注最关键的 3-4 个文件，其余文件简要提及即可 策略二：@ 符号快速引用 在对话中使用 @ 符号直接引用文件或目录，比描述文件路径更高效：</p>

<p># 引用单个文件 解释 @src/utils/auth.js 中的 JWT 验证逻辑 # 对比两个文件 比较 @src/auth.js 和 @src/auth-v2.js 的实现差异， 判断 v2 版本是否可以安全替换 v1 # 引用目录获取结构 @src/components 有哪些组件？按功能分类列出 策略三：子代理调查 子代理在独立的上下文窗口中运行，调查完毕后向主会话汇报摘要，不污染主上下文。适合需要大量文件扫描 的调查任务：</p>

<p>使用子代理调查我们的认证系统如何处理 token 刷新， 以及是否存在可以复用的 OAuth 工具类。</p>

<p>只汇报结论和相关文件路径，不需要展示中间过程。</p>

<p>找到 PaymentService.calculateTax() 的所有调用方， 分析哪些调用没有正确处理税率精度问题。</p>

<p>导航的反模式（避免这些） 注意 不要让 Claude 用 Bash 的 cat 、find 、grep 命令代替内置工具。Read/Glob/Grep 的权限系统经过优化， 体验更好，也能避免潜在的权限问题。每次你看到 Claude 用 cat filename，都可以提醒它用 Read 工具代 替。</p>

<p>反模式 | 问题 | 正确做法</p>

<p>直接 Read 整个大文件 | 消耗大量上下文，性能下降 | 先 Grep 定位，再分段 Read</p>

<p>用 Bash cat 读文件 | 权限处理较差，体验欠佳 | 使用 Read 工具</p>

<p>用 Bash find 找文件 | 结果无排序，权限问题 | 使用 Glob 工具</p>

<p>一次读取 10 个以上文件 | 上下文快速耗尽 | 分批读取，优先读最相关的</p>

<p>不指定搜索范围 | Claude 漫无目的扫描 | 明确指定目录或文件类型</p>
