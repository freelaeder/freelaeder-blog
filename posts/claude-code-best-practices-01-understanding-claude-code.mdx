## 笔记整理

<p>Claude Code 是 Anthropic 推出的 agentic 编程工具，以命令行 CLI 的形式运行在你的终端中。它不 是一个对话助手，而是一个能够自主探索代码库、执行命令、修改文件、管理 Git 的 AI 代理。理解它 的运行机制，是高效使用它的第一步。</p>

<p>什么是 Claude Code Claude Code 由 Anthropic 官方发布，截至 2026 年 3 月已演进到 v2.1.75+，默认模型为 Opus 4.6，支持最高 100 万 token 的上下文窗口（Max / Team / Enterprise 订阅）。它不通过浏览器运行，而是作为一个本地进程驻 留在你的终端，与项目文件系统直接交互。</p>

<p>从本质上说，Claude Code 是一个具身化 AI 代理（Embodied AI Agent）：它能感知环境（读取文件和命令输 出），做出决策（选择哪个工具、如何修改代码），并执行动作（写文件、运行脚本、提交 Git）。这与你在网页 上和 Claude 聊天是本质不同的两种体验。</p>

<p>一句话定义：Claude Code 是活在你终端里的 AI 工程师搭档，它可以阅读你的全部代码库，自主规划并执行多 步任务，而不只是回答你的问题。</p>

<p>与普通 Claude（Claude.ai）的核心区别 很多人第一次使用 Claude Code 时，会按照使用 Claude.ai 的习惯来提问——粘贴一段代码，让它解释或修 改。这种方式虽然有效，但远远没有发挥出 Claude Code 的真实能力。要理解两者的根本差异，可以看下面这 张对比表：</p>

<p>维度普通 Claude（Claude.ai） Claude Code</p>

<p>交互方式 | 浏览器网页对话 | 终端 CLI，集成于开发环境</p>

<p>文件访问 | 需手动粘贴代码或上传文件 | 直接读写本地文件系统</p>

<p>代码执行 | 无法执行代码 | 可执行任意 Shell 命令</p>

<p>自主性 | 单轮问答，被动响应 | Agentic 循环，主动探索、计划、执行</p>

<p>代码库理解 | 只看到你提供的代码片段 | 理解整个项目结构与依赖</p>

<p>Git 集成 | 仅能生成 git 命令文本 | 直接执行 git 操作，创建 PR</p>

<p>持久记忆 | 仅限当前对话（Projects 功能有限记忆） | CLAUDE.md + Auto Memory 跨会话持久化</p>

<p>上下文管理 | 手动管理，容易遗漏 | 自动压缩，支持会话恢复与分支</p>

<p>并行工作手动多标签页 Git Worktrees + 并行 Agent Teams 最关键的区别是自主性。Claude.ai 等待你给出完整的指令，然后给出一个响应。Claude Code 则会主动探索 ——它会先读取相关文件，理解上下文，然后制定计划，分多步执行，遇到问题还会自行调试。</p>

<p>Agentic 循环——核心运行架构 Claude Code 的运行机制基于一个三阶段的 Agentic 循环（Agentic Loop）。每当你提出一个任务，Claude Code 不会立即给出答案，而是进入这个循环：</p>

<p>阶段一 阶段二 阶段三</p>

<p>收集上下文 | →</p>

<p>采取行动 | →</p>

<p>验证结果 | → | 重复 / 完成</p>

<p>1 | 收集上下文（Gather Context）</p>

<p>Claude Code 首先使用只读工具探索环境：读取相关文件（Read）、搜索代码（Grep）、查找文件结构 （Glob）、运行查询命令了解当前状态。它会在行动之前尽可能充分地理解问题域，就像一个有经验的工 程师在动手改代码前会先把相关代码都看一遍。</p>

<p>2 | 采取行动（Take Action）</p>

<p>在充分理解上下文之后，Claude Code 开始执行：编辑文件（Edit / Write）、执行脚本（Bash）、提交代 码（git commit）、创建 Pull Request。每一个行动都是基于上一步收集到的信息做出的有依据的决策， 而非盲目执行。</p>

<p>3 | 验证结果（Verify Results）</p>

<p>执行之后，Claude Code 会主动验证：运行测试（pytest / npm test）、检查命令输出、对比前后差异。</p>

<p>如果测试失败，它会重新进入循环，分析失败原因，修改代码，再次验证。这个&quot;写代码 → 测试 → 修 bug → 再测试&quot;的内循环完全由 Claude Code 自主完成。</p>

<p>核心建议 这个循环使 Claude Code 能够处理跨越多个文件、多个步骤的复杂任务。你可以在任意时刻打断它（按 Esc ），审查当前状态，然后继续或调整方向。</p>

<p>理解 Agentic 循环的意义在于：你的任务描述越清晰，Claude Code 就能越精准地进入正确的循环轨道。模糊 的指令会导致它在探索阶段消耗过多上下文，甚至走错方向。后续章节会详细讲解如何撰写高质量的任务提 示。</p>

<p>安装与初始化 Claude Code 通过 npm 全局安装，无需配置复杂环境：</p>

<p># 安装（需要 Node.js 18+） # 在项目目录启动 # 首次运行会引导你完成 Anthropic 账号授权 注意 Claude Code 需要 Anthropic API 访问权限（Pro / Max / Team / Enterprise 订阅）。免费版 Claude.ai 账号无 法使用 Claude Code。</p>

<p>启动后，你会看到一个简洁的终端界面。Claude Code 会自动检测你的项目类型（package.json、 pyproject.toml 等），读取项目根目录的 CLAUDE.md 文件（如果存在），初始化会话上下文。</p>
